# Security Policy

## Reporting a Vulnerability

Please do not file public issues for security vulnerabilities.

This repository does not currently publish a dedicated security email. Contact
the maintainers privately through GitHub. If you cannot find a private contact
path, open a public issue asking for a security contact and do not include
vulnerability details.

Include enough detail to reproduce or understand the issue, including affected
package names, versions or commits, and any relevant browser/runtime details.

## Response

Prism is early-stage software. Maintainers will make a good-faith effort to
acknowledge security reports promptly, investigate them privately, and publish
fixes or mitigations when appropriate.
