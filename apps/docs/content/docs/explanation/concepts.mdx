Prism is not a canvas framework, game engine, charting library, or editor framework.

Prism is runtime infrastructure for apps that want real HTML/CSS elements to participate as managed canvas surfaces.

## Platform and runtime

HTML-in-Canvas is the platform capability. Prism builds on that proposal and the related WICG work.

Prism provides the runtime layer:

- register HTML elements as surfaces
- size the backing store
- keep public bounds in CSS pixels
- schedule and execute paint
- draw surfaces in the app's chosen order
- deactivate undrawn surfaces
- clean up registered elements
- provide `paintOnce()` for readiness

## App code still matters

Your app still owns its product model. Prism does not decide your templates, transforms, data, charting rules, editor state, or interaction model.

That is why Prism examples look different:

- Atlantic uses Prism around a canvas data visualization.
- React Composer Lite uses Prism around a composition/editor workflow.
