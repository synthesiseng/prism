---
title: Coordinate Spaces
description: Surface bounds and input coordinates use CSS pixels; direct canvas drawing uses backing-store pixels.
---

Prism keeps the public boundary in CSS pixels.

## CSS pixels

Surface bounds use CSS pixels:

```ts
const surface = runtime.registerSurface(element, {
  bounds: { x: 24, y: 24, width: 320, height: 180 }
});
```

Pointer conversion also returns CSS-space coordinates:

```ts
canvas.addEventListener("pointermove", (event) => {
  const point = runtime.clientToCanvasPoint(event.clientX, event.clientY);
});
```

## Canvas backing-store pixels

Direct drawing inside `onPaint()` uses the canvas backing store.

Use runtime helpers when you need to align manual drawing with CSS-space surface geometry:

```ts
runtime.onPaint(({ ctx, drawSurface }) => {
  drawSurface(surface);

  const size = runtime.cssLengthToCanvasPixels(24);
  ctx.fillRect(0, 0, size, size);
});
```

Apps should not multiply by `devicePixelRatio` directly.
