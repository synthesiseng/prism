---
title: Use HTML Surfaces in Canvas Data Viz
description: Combine canvas-rendered data with Prism-managed HTML panels, legends, and tooltips.
---

Use canvas for dense marks and HTML/CSS for interface surfaces.

```ts
const runtime = new CanvasRuntime(canvas, { backend: "auto" });

const legend = runtime.registerSurface(legendElement, {
  bounds: { x: 24, y: 420, width: 240, height: 180 }
});

const tooltip = runtime.registerSurface(tooltipElement, {
  bounds: { x: 0, y: 0, width: 220, height: 96 }
});

runtime.onPaint(({ ctx, drawSurface }) => {
  drawChart(ctx);

  drawSurface(legend);

  if (hoveredPoint) {
    tooltip.setBounds({
      x: hoveredPoint.x + 12,
      y: hoveredPoint.y + 12,
      width: 220,
      height: 96
    });
    drawSurface(tooltip);
  }
});
```

The app owns the data, projection, hit testing, and UI content. Prism owns registration, paint ordering, bounds, invalidation, and readiness.
